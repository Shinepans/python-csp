Class FileChannel

Channel objects using files on disk.

FileChannel objects close their files after each read or write operation. The advantage of this is that client code can create as many FileChannel objects as it wishes (unconstrained by the operating system's maximum number of open files). In return there is a performance hit -- reads and writes are around 10 x slower on FileChannel objects compared to Channel objects.
